---
title: Renderer Registries
description: Resolve JSON field definitions to framework components by component name or field type.
icon: Boxes
---

Renderer registries are the boundary between stored data and UI code. A field can store `component: 'email-input'`; the app decides which component that name resolves to.

[Open the renderer registry source on GitHub](https://github.com/VueEasyForm/tgb-form/blob/main/packages/core/src/renderer-registry.ts).

## What

A registry maps the renderer components you already defined to stable names and field-type fallbacks:

```ts twoslash
import { createRendererRegistry, FieldDataType } from '@tgb-form/core';

const TextInput = { kind: 'TextInput' };
const EmailInput = { kind: 'EmailInput' };
const NumberInput = { kind: 'NumberInput' };
const CheckboxInput = { kind: 'CheckboxInput' };

export const renderers = createRendererRegistry({
  byName: {
    'email-input': EmailInput,
    'short-text': TextInput,
  },
  byType: {
    [FieldDataType.String]: TextInput,
    [FieldDataType.Number]: NumberInput,
    [FieldDataType.Boolean]: CheckboxInput,
  },
});
```

React and Vue adapters expose framework-specific helpers with the same shape:

```ts twoslash
import { createReactRendererRegistry } from '@tgb-form/react';
import { createVueRendererRegistry } from '@tgb-form/vue';
```

## Why

Definitions need to stay serializable. Components, hooks, and closures cannot be serialized. Registry lookup keeps code at runtime while letting saved form data name the UI it needs.

## How

Resolve a field with `resolveRenderer`:

```ts twoslash
import { defineForm, FieldDataType, resolveRenderer } from '@tgb-form/core';

const TextInput = { kind: 'TextInput' };
const renderers = {
  byName: {
    'email-input': TextInput,
  },
  byType: {
    [FieldDataType.String]: TextInput,
  },
};

const form = defineForm(
  {
    fields: {
      email: {
        type: FieldDataType.String,
        defaultValue: '',
        label: 'Email',
        component: 'email-input',
        props: { autocomplete: 'email' },
      },
    },
  },
  { renderers },
);

const Renderer = resolveRenderer(form.fields.email, renderers);
```

Resolution is strict:

| Field definition              | Resolver behavior                              |
| ----------------------------- | ---------------------------------------------- |
| Has `component`               | Look up `registry.byName[component]`.          |
| Has unknown `component`       | Throw `Missing renderer named "..."`.          |
| Has no `component`            | Fall back to `registry.byType[field.type]`.    |
| Has no matching type renderer | Throw `Missing renderer for field type "..."`. |

When you pass a registry to `defineForm`, TypeScript checks `component` against the registry's `byName` keys.

## Registry Shape

Renderer registries are plain lookup tables:

```ts twoslash
import { createRendererRegistry, FieldDataType } from '@tgb-form/core';

const TextInput = { kind: 'TextInput' };
const EmailInput = { kind: 'EmailInput' };

const renderers = createRendererRegistry({
  byName: {
    'email-input': EmailInput,
  },
  byType: {
    [FieldDataType.String]: TextInput,
  },
});

renderers.byName['email-input']; // EmailInput
renderers.byType[FieldDataType.String]; // TextInput
```

Use the framework helpers when the registry stores React or Vue components:

```ts twoslash
import { createReactRendererRegistry } from '@tgb-form/react';
import { createVueRendererRegistry } from '@tgb-form/vue';
```

These helpers call the core registry factory while preserving framework-specific renderer types.
