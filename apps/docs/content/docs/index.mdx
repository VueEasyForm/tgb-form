Define forms as data. Render them anywhere.

TGB Form is a fast, flexible wrapper around [TanStack Form](https://tanstack.com/form) and [Valibot](https://valibot.dev). It keeps form schemas JSON-safe while leaving runtime code free to choose React, Vue, custom renderers, storage, and validation extensions.

## What

TGB Form gives you a portable form definition:

```ts twoslash
import { defineForm, FieldDataType, ValidationRuleKind } from '@tgb-form/core';

export const newsletterForm = defineForm({
  fields: {
    email: {
      type: FieldDataType.String,
      defaultValue: '',
      label: 'Email',
      component: 'email-input',
      props: { placeholder: 'you@example.com', autocomplete: 'email' },
      rules: [
        { kind: ValidationRuleKind.Required, message: 'Email is required' },
        { kind: ValidationRuleKind.Email, message: 'Enter a valid email' },
      ],
    },
    subscribed: {
      type: FieldDataType.Boolean,
      defaultValue: true,
      label: 'Subscribe',
    },
  },
});
```

The same definition can be serialized, restored, compiled to Valibot, converted to TanStack Form options, and rendered through a registry.

## Why

Most form libraries start from components. That works until the form needs to be stored, edited in a builder, generated from configuration, or rendered in more than one framework.

TGB Form starts from data:

| Need                   | TGB Form layer      |
| ---------------------- | ------------------- |
| Save a form definition | `serializeForm`     |
| Load builder output    | `deserializeForm`   |
| Validate values        | `toValibotSchema`   |
| Use TanStack Form      | `toTanStackOptions` |
| Choose UI components   | renderer registries |

TanStack Form still owns form state. Valibot still owns validation. TGB Form connects them with a schema that can move through storage.

## How

### Install the core packages [step]

```package-install
@tgb-form/core valibot @tanstack/form-core
```

### Install the adapters [step]

#### React adapter

```package-install
@tgb-form/react @tanstack/react-form
```

#### Vue adapter

```package-install
@tgb-form/vue @tanstack/vue-form
```

<Callout type="idea">
  The design of TGB Form actually allows for integrations for other frameworks too, thanks to the
  framework-agnostic core of TanStack Form and Valibot! But we prioritize these 2 frameworks first
  for our internal usages. If you want to contribute an adapter for your loved framework, feel free
  to open a PR or an issue on GitHub!
</Callout>

### Render (in React): [step]

```tsx twoslash
import { defineForm, FieldDataType, ValidationRuleKind } from '@tgb-form/core';
import { BaseReactRendererProps, TgbForm, createReactRendererRegistry } from '@tgb-form/react';

const newsletterForm = defineForm({
  fields: {
    email: {
      type: FieldDataType.String,
      defaultValue: '',
      label: 'Email',
      component: 'email-input',
      props: { placeholder: 'you@example.com', autocomplete: 'email' },
      rules: [
        { kind: ValidationRuleKind.Required, message: 'Email is required' },
        { kind: ValidationRuleKind.Email, message: 'Enter a valid email' },
      ],
    },
  },
});

function TextInput({ field, label, name, props, value, errors }: BaseReactRendererProps) {
  return (
    <label>
      <span>{label}</span>
      <input
        name={name}
        value={String(value ?? '')}
        placeholder={String(props?.placeholder ?? '')}
        onBlur={() => field.handleBlur()}
        onChange={(event) => field.handleChange(event.currentTarget.value)}
      />
      {errors.length > 0 ? <small>{errors.map(String).join(', ')}</small> : null}
    </label>
  );
}

const renderers = createReactRendererRegistry({
  byName: { 'email-input': TextInput },
  byType: { [FieldDataType.String]: TextInput },
});

export function NewsletterForm() {
  return (
    <TgbForm
      definition={newsletterForm}
      renderers={renderers}
      tanstackOptions={{
        onSubmit: async ({ value }) => {
          console.log(value);
        },
      }}
    />
  );
}
```

<Callout>
  It's actually very similar to do so in Vue, and that's the strength of our library! We don't show
  the example here because it would look too similar to each other and would make you confused
  instead. Just swap around the framework-specific parts and it will work right away!
</Callout>

### Store and restore the definition: [step]

```ts twoslash
import {
  defineForm,
  deserializeForm,
  FieldDataType,
  serializeForm,
  toTanStackOptions,
} from '@tgb-form/core';

const newsletterForm = defineForm({
  fields: {
    email: {
      type: FieldDataType.String,
      defaultValue: '',
    },
  },
});

const stored = JSON.stringify(serializeForm(newsletterForm));
const restored = deserializeForm(stored);
const options = toTanStackOptions(restored);
```
