EasyForm turns form definitions into portable data. Author fields once, serialize them as JSON, validate them with Valibot, and hand defaults plus submit validation to TanStack Form.

## Install

```sh
pnpm add @easyform/core valibot @tanstack/form-core
```

Add a framework adapter package when you are wiring renderers into an app:

```sh
pnpm add @easyform/react
pnpm add @easyform/vue
```

## Newsletter Form

[Open this example on GitHub](https://github.com/VueEasyForm/easyform/blob/main/packages/core/tests/index.test.ts).

```ts
import { defineForm, FieldDataType, toTanStackOptions, ValidationRuleKind } from '@easyform/core';

export const newsletterForm = defineForm({
  fields: {
    email: {
      type: FieldDataType.String,
      defaultValue: '',
      label: 'Email',
      description: 'Where should we send product updates?',
      component: 'email-input',
      props: { autocomplete: 'email', placeholder: 'you@example.com' },
      rules: [
        { kind: ValidationRuleKind.Required, message: 'Email is required' },
        { kind: ValidationRuleKind.Email, message: 'Enter a valid email address' },
      ],
    },
    subscribed: {
      type: FieldDataType.Boolean,
      defaultValue: true,
      label: 'Subscribe me',
    },
  },
  meta: {
    submitLabel: 'Join newsletter',
  },
});

export const tanstackOptions = toTanStackOptions(newsletterForm, {
  formId: 'newsletter',
});
```

## What EasyForm Gives You

| Feature                 | API                                         |
| ----------------------- | ------------------------------------------- |
| Normalize definitions   | `defineForm`                                |
| Persist definitions     | `serializeForm`, `deserializeForm`          |
| Compile validation      | `toValibotSchema`                           |
| Create TanStack options | `toTanStackOptions`, `getDefaultValues`     |
| Resolve UI components   | `createRendererRegistry`, `resolveRenderer` |
| Attach code validators  | `createValidatorRegistry`                   |
