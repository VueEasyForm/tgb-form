`@easyform/vue` connects portable definitions to Vue components. The definition stays JSON-safe; renderers stay in application code.

[Open the Vue package on GitHub](https://github.com/VueEasyForm/easyform/tree/main/packages/vue).

## Supported Flows

`EzForm` supports three usage levels:

1. Implicit: `EzForm` creates the TanStack form instance and reads renderers from `EzFormProvider`.
2. Hybrid: you provide either the TanStack form instance or the renderer registry, while `EzForm` supplies the rest.
3. Fully explicit: you provide both `instance` and `renderers` directly to `EzForm`.

## Quick Start

```vue
<script setup lang="ts">
  import { defineForm, FieldDataType, ValidationRuleKind } from '@easyform/core';
  import { EzForm, EzFormProvider, createVueRendererRegistry } from '@easyform/vue';
  import CheckboxInput from './CheckboxInput.vue';
  import TextInput from './TextInput.vue';

  const definition = defineForm({
    fields: {
      email: {
        type: FieldDataType.String,
        defaultValue: '',
        label: 'Email',
        component: 'email-input',
        props: { placeholder: 'you@example.com' },
        rules: [{ kind: ValidationRuleKind.Email, message: 'Enter a valid email' }],
      },
      subscribed: {
        type: FieldDataType.Boolean,
        defaultValue: true,
        label: 'Subscribe',
      },
    },
  });

  const renderers = createVueRendererRegistry({
    byName: {
      'email-input': TextInput,
    },
    byType: {
      [FieldDataType.String]: TextInput,
      [FieldDataType.Boolean]: CheckboxInput,
    },
  });
</script>

<template>
  <EzFormProvider :renderers="renderers">
    <EzForm
      :definition="definition"
      :tanstack-options="{ onSubmit: async ({ value }) => console.log(value) }"
    />
  </EzFormProvider>
</template>
```

That is the implicit flow: application code provides a renderer registry once through `EzFormProvider`, and `EzForm` creates the TanStack form instance from `definition` and `tanstackOptions`.

## What EzForm Does

`EzForm`:

- Calls `toTanStackOptions(definition, tanstackOptions)` when no `instance` is passed.
- Creates a TanStack Vue Form instance with `useForm`.
- Provides the form instance to child fields.
- Renders fields by `order`, then declaration order.
- Calls `form.handleSubmit()` on submit.

`EzField` is the automatic registry-based resolver. It wraps TanStack's field component, resolves the renderer for one field definition, and passes TanStack field bindings and state into that renderer.

Pass `fields` to render a named subset:

```vue
<EzForm :definition="definition" :fields="['email']" :renderers="renderers" />
```

## Why

Vue apps usually want renderer components to look like normal Vue components. `@easyform/vue` keeps that style intact:

- renderers can be ordinary `.vue` SFCs
- app-wide registries can live in provider-based composition
- `EzForm` handles the TanStack field wiring without forcing render functions into application code

The adapter still uses `defineComponent` internally where it makes sense, but consumer-facing examples should stay close to how Vue teams already write components.

## How

### Hybrid Flow

Pass local renderers while still letting `EzForm` create the TanStack form instance:

```vue
<EzForm
  :definition="definition"
  :renderers="renderers"
  :tanstack-options="{ onSubmit: async ({ value }) => console.log(value) }"
/>
```

Or pass an existing TanStack form instance while still reading renderers from the provider:

```vue
<script setup lang="ts">
  import { toTanStackOptions } from '@easyform/core';
  import { useForm } from '@tanstack/vue-form';

  const form = useForm(
    toTanStackOptions(definition, {
      onSubmit: async ({ value }) => {
        console.log(value);
      },
    }),
  );
</script>

<template>
  <EzFormProvider :renderers="renderers">
    <EzForm
      :definition="definition"
      :instance="form"
    />
  </EzFormProvider>
</template>
```

### Fully Explicit Flow

Pass an existing TanStack form instance when you want full control:

```vue
<script setup lang="ts">
  import { toTanStackOptions } from '@easyform/core';
  import { useForm } from '@tanstack/vue-form';

  const form = useForm(
    toTanStackOptions(definition, {
      onSubmit: async ({ value }) => {
        console.log(value);
      },
    }),
  );
</script>

<template>
  <EzForm
    :definition="definition"
    :instance="form"
    :renderers="renderers"
  />
</template>
```

## Renderer Props

Every Vue renderer receives:

| Prop          | Description                                         |
| ------------- | --------------------------------------------------- |
| `name`        | Field name from the definition map.                 |
| `field`       | TanStack field binding.                             |
| `form`        | Current TanStack form instance.                     |
| `label`       | Field label, if present.                            |
| `description` | Field description, if present.                      |
| `props`       | JSON-safe renderer props from the field definition. |
| `value`       | Current field value.                                |
| `errors`      | Current field errors.                               |

Renderer components should translate these props into the input components used by your app.

## Provider And Overrides

Use `EzFormProvider` for app-wide renderers:

```vue
<EzFormProvider :renderers="renderers">
  <EzForm :definition="definition" />
</EzFormProvider>
```

Use the `renderers` prop for a local override:

```vue
<EzForm :definition="definition" :renderers="localRenderers" />
```

Renderer resolution follows core rules: explicit `component` names resolve through `byName`; fields without `component` fall back to `byType[field.type]`.
