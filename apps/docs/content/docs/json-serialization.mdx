EasyForm definitions are designed to survive a JSON round trip. The stored value contains data. Runtime registries provide code after the value is loaded.

[Open serialization source on GitHub](https://github.com/VueEasyForm/easyform/blob/main/packages/core/src/schema/form.ts).

## What

```ts twoslash
import {
  defineForm,
  deserializeForm,
  FieldDataType,
  serializeForm,
  ValidationRuleKind,
} from '@easyform/core';

const form = defineForm({
  fields: {
    email: {
      type: FieldDataType.String,
      defaultValue: '',
      label: 'Email',
      component: 'email-input',
      props: { autocomplete: 'email' },
      rules: [{ kind: ValidationRuleKind.Required }, { kind: ValidationRuleKind.Email }],
    },
  },
});

const stored = JSON.stringify(serializeForm(form));
const restored = deserializeForm(stored);
```

`serializeForm` returns JSON-safe form data. `deserializeForm` parses stored data and normalizes it with `defineForm`.

## Why

Form builders need durable output. A saved definition should work in local storage, a database row, a CMS field, an API response, or a file.

Renderer and validator registries are code, so they are not serialized. Attach them when defining or deserializing a form. Core only uses validator registries when compiling validation; renderer registries stay available for adapter code and for component-name type narrowing during authoring.

## How

```ts twoslash
import {
  createRendererRegistry,
  createValidatorRegistry,
  deserializeForm,
  FieldDataType,
} from '@easyform/core';
import * as v from 'valibot';

const TextInput = (props: unknown) => props;
const CheckboxInput = (props: unknown) => props;

const renderers = createRendererRegistry({
  byType: {
    [FieldDataType.String]: TextInput,
    [FieldDataType.Boolean]: CheckboxInput,
  },
});

const validators = createValidatorRegistry().register('reservedName', ({ message }) =>
  v.check((value: string) => !['admin', 'root'].includes(value), message),
);

const storedJson = JSON.stringify({
  fields: {
    name: {
      type: FieldDataType.String,
      defaultValue: '',
      rules: [],
    },
  },
});

const restored = deserializeForm(storedJson, {
  renderers,
  validators,
});
```

## Stored Shape

```json
{
  "fields": {
    "name": {
      "type": "string",
      "defaultValue": "",
      "label": "Name",
      "rules": [{ "kind": "required" }]
    },
    "contact": {
      "type": "object",
      "defaultValue": { "email": "" },
      "fields": {
        "email": {
          "type": "string",
          "defaultValue": "",
          "label": "Email",
          "rules": [{ "kind": "email" }]
        }
      }
    },
    "tags": {
      "type": "array",
      "defaultValue": [],
      "element": {
        "type": "string",
        "defaultValue": ""
      }
    }
  }
}
```

## What Can Be Serialized

| Property                         | JSON-safe |
| -------------------------------- | --------- |
| `fields`                         | yes       |
| `meta`                           | yes       |
| `label`, `description`, `order`  | yes       |
| `component`, `props`             | yes       |
| `rules`, `validators` references | yes       |
| Renderer registry entries        | no        |
| Validator compiler functions     | no        |
