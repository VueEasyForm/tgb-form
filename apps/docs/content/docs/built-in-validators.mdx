Built-in validators are validation rules that can be stored with a form definition. TGB Form compiles them into Valibot through `toValibotSchema`; `toTanStackOptions` places the same schema under `validators.onSubmit`.

[Open validation tests on GitHub](https://github.com/VueEasyForm/easyform/blob/main/packages/core/tests/index.test.ts).

## What

```ts twoslash
import * as v from 'valibot';
import { defineForm, FieldDataType, toValibotSchema, ValidationRuleKind } from '@tgb-form/core';

const accountForm = defineForm({
  fields: {
    name: {
      type: FieldDataType.String,
      defaultValue: '',
      rules: [
        { kind: ValidationRuleKind.Required, message: 'Name is required' },
        { kind: ValidationRuleKind.MinLength, value: 2, message: 'Name is too short' },
      ],
    },
    age: {
      type: FieldDataType.Number,
      defaultValue: 18,
      rules: [{ kind: ValidationRuleKind.Min, value: 18, message: 'Must be an adult' }],
    },
    website: {
      type: FieldDataType.String,
      defaultValue: '',
      rules: [{ kind: ValidationRuleKind.Url, message: 'Enter a valid URL' }],
    },
  },
});

const schema = toValibotSchema(accountForm);
const result = v.safeParse(schema, {
  name: 'Ada',
  age: 28,
  website: 'https://example.com',
});
```

## Why

Common validation should travel with stored forms. Built-in validators keep required fields, length checks, numeric bounds, patterns, and string formats in JSON-safe data.

Use [Custom Validators](/docs/custom-validators) when a rule needs runtime code.

## Rules

| Rule        | Value               | Applies to                                  |
| ----------- | ------------------- | ------------------------------------------- |
| `Required`  | none                | strings, arrays, objects, numbers, booleans |
| `MinLength` | number              | strings and arrays                          |
| `MaxLength` | number              | strings and arrays                          |
| `Pattern`   | string regex source | strings                                     |
| `Min`       | number              | numbers                                     |
| `Max`       | number              | numbers                                     |
| `Email`     | none                | strings                                     |
| `Url`       | none                | strings                                     |

## Example

```ts twoslash
import { defineForm, FieldDataType, ValidationRuleKind } from '@tgb-form/core';

export const profileForm = defineForm({
  fields: {
    username: {
      type: FieldDataType.String,
      defaultValue: '',
      label: 'Username',
      rules: [
        { kind: ValidationRuleKind.Required, message: 'Choose a username' },
        { kind: ValidationRuleKind.MinLength, value: 3, message: 'Use at least 3 characters' },
        {
          kind: ValidationRuleKind.Pattern,
          value: '^[a-z0-9_]+$',
          message: 'Use lowercase letters, numbers, and underscores',
        },
      ],
    },
  },
});
```

The rule order is preserved when the Valibot schema is built.
