A form definition is the data TGB Form moves through the system. It contains fields, defaults, labels, validation rules, renderer hints, and JSON-safe metadata.

[Open the definition source on GitHub](https://github.com/VueEasyForm/tgb-form/blob/main/packages/core/src/schema/definitions.ts).

## What

Each field has a `type` and a matching `defaultValue`:

```ts twoslash
import { defineForm, FieldDataType } from '@tgb-form/core';

export const profileForm = defineForm({
  fields: {
    displayName: {
      type: FieldDataType.String,
      defaultValue: '',
      label: 'Display name',
      description: 'Shown on your public profile.',
      order: 1,
      component: 'text-input',
      props: { placeholder: 'Ada Lovelace' },
      meta: { section: 'identity' },
    },
    age: {
      type: FieldDataType.Number,
      defaultValue: 18,
      label: 'Age',
      order: 2,
    },
    publicProfile: {
      type: FieldDataType.Boolean,
      defaultValue: false,
      label: 'Public profile',
      order: 3,
    },
  },
  meta: {
    title: 'Profile',
  },
});
```

## Why

Definitions are plain data so they can be stored and moved before they are rendered. A builder can save the object. A server can return it. React and Vue can render it through different registries.

`component` and `props` are renderer hints, not framework components:

```ts twoslash
import { FieldDataType } from '@tgb-form/core';

const field = {
  type: FieldDataType.String,
  defaultValue: '',
  component: 'email-input',
  props: {
    autocomplete: 'email',
    placeholder: 'you@example.com',
  },
};
```

The app decides what `email-input` means at runtime.

## How

Use the built-in data types:

| Type                    | Default value must be |
| ----------------------- | --------------------- |
| `FieldDataType.String`  | `string`              |
| `FieldDataType.Number`  | `number`              |
| `FieldDataType.Boolean` | `boolean`             |
| `FieldDataType.Object`  | JSON object           |
| `FieldDataType.Array`   | JSON array            |

`defineForm` validates defaults while normalizing the definition. A number field with a string default throws before the form reaches a renderer.

## Nested Fields

Objects use `fields`. Arrays use `element`.

```ts twoslash
import { defineForm, FieldDataType, ValidationRuleKind } from '@tgb-form/core';

export const profileWithContact = defineForm({
  fields: {
    name: {
      type: FieldDataType.String,
      defaultValue: '',
      label: 'Name',
      rules: [{ kind: ValidationRuleKind.Required }],
    },
    contact: {
      type: FieldDataType.Object,
      defaultValue: { email: '', phone: '' },
      label: 'Contact',
      fields: {
        email: {
          type: FieldDataType.String,
          defaultValue: '',
          label: 'Email',
          rules: [{ kind: ValidationRuleKind.Email }],
        },
        phone: {
          type: FieldDataType.String,
          defaultValue: '',
          label: 'Phone',
        },
      },
    },
    tags: {
      type: FieldDataType.Array,
      defaultValue: [],
      label: 'Tags',
      element: {
        type: FieldDataType.String,
        defaultValue: '',
      },
    },
  },
});
```

Nested definitions compile recursively for built-in validation rules.
